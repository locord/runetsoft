Инкрементация на странице показа банера
Как вариант рассмотрел бы Redis для хранения:
- Есть счетчик из коробки, https://redis.io/commands/incr
- Атомарный, не нужны блокировки
- Быстрый, так как работае в памяти
- Умеет сохранять данные на диск

Для уникальности можно хранить ключ для инкремента в функции
/**
$banner - объект баннера
$user - объект пользователя, если показываются для авторизованного пользователя
*/
public function showedBannerCacheKey($banner, $user)
{
    return sprintf("users.%s.showed.%s", $user->id, $banner->id);
}
Также можно использовать cookies для идентификации браузера, но нужно будет следить за корректностью принимаемых данных,
т.к куки можно легко подделать или вовсе октлючить, файлы, также не обладают гибкостью.
Можно храниь и в базе, если нет большой нагрузки на запись.
Redis обеспечивает быструю выборку, сохранение и дальнейшую обработку,
например удобное ранжирование по кол-ву показов баннеров, если необходим учет нескольких баннеров и строим полноценную
банерную систему
